# sufdo Development Plan

## 📅 День 1 (v3.1.0) - Silent/Verbose режимы
- [ ] `--silent` - тихий режим (только ошибки)
- [ ] `--verbose` - подробный вывод с отладкой
- [ ] `--quiet` - alias для silent
- [ ] `--debug` - режим отладки
- [ ] `--trace` - трассировка выполнения

## 📅 День 2 (v3.2.0) - Safety режимы
- [ ] `--dry-run` - показать команду без выполнения
- [ ] `--confirm` - спрашивать подтверждение
- [ ] `--safe-mode` - запрет деструктивных команд
- [ ] `--no-destructive` - блокировка rm -rf и т.д.
- [ ] `--backup` - резервное копирование перед выполнением

## 📅 День 3 (v3.3.0) - Логирование
- [ ] `--log` - логировать в файл
- [ ] `--log-file` - указать файл лога
- [ ] `--log-level` - уровень логирования
- [ ] `--syslog` - отправка в syslog
- [ ] `--rotate-logs` - ротация логов

## 📅 День 4 (v3.4.0) - Уведомления
- [ ] `--notify` - toast уведомление после выполнения
- [ ] `--notify-sound` - со звуком
- [ ] `--discord` - webhook в Discord
- [ ] `--telegram` - уведомление в Telegram
- [ ] `--email` - отправка на email

## 📅 День 5 (v3.5.0) - Статистика
- [ ] `--stats` - статистика использования
- [ ] `--top` - топ самых частых команд
- [ ] `--time` - время выполнения команд
- [ ] `--success-rate` - процент успешных команд
- [ ] `--export-stats` - экспорт в JSON/CSV
